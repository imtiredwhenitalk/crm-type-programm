## Швидкий старт

### Вимоги

- Python **3.8** або новіший
- Стандартна бібліотека Python (без зовнішніх залежностей)
- `tkinter` — входить до стандартного пакету Python

### Встановлення та запуск

```bash
# Клонувати репозиторій
git clone https://github.com/your-username/crm-system.git
cd crm-system

# Запустити
python crm.py
```

> **Windows:** якщо Tkinter не встановлений — при інсталяції Python позначте галочку «tcl/tk and IDLE».

> **Linux (Ubuntu/Debian):**
> ```bash
> sudo apt-get install python3-tk
> ```

### Перший вхід

| Поле | Значення |
|---|---|
| Логін | `admin` |
| Пароль | `admin` |

> ⚠️ Після першого входу рекомендується змінити пароль: **Профіль → Змінити пароль**.

## Модулі

### `crm.py` — точка входу

Ініціалізує базу даних (`db_init()`) та відкриває вікно авторизації (`LoginWindow`). Весь код виконується тут лише один раз при старті.

---

### `db/` — шар даних

#### `db.py`
Центральний модуль роботи з SQLite. Відповідає за:
- Підключення до `crm.db` з `PRAGMA journal_mode=WAL` для надійності
- Ініціалізацію схеми при першому запуску
- Автоматичні міграції (додавання нових колонок без втрати даних)
- Хешування паролів (`hash_pw` — SHA-256)

**Таблиці:** `users`, `workers`, `tasks`, `logs`, `time_sessions`, `notes`, `login_attempts`

#### `modalbase.py`
Базовий клас `Modal` для всіх модальних вікон. Ключова особливість — **скролювана область вмісту** (`Canvas` + `Scrollbar`), завдяки якій кнопка «Зберегти» завжди видима незалежно від висоти форми. Надає методи:
- `field(label, ...)` — текстове поле з підписом
- `combobox(label, values, ...)` — випадаючий список
- `textarea(label, ...)` — багаторядковий текст
- `add_ok(text, cmd)` — кнопки збереження/скасування у фіксованому футері

#### `current_user.py`
Єдиний глобальний об'єкт `current_user` — словник з даними авторизованого користувача (`id`, `username`, `role`, `phone`, `email`, `org`). Доступний з будь-якого модуля через імпорт.

#### `session.py`
Управління збереженою сесією:
- `save_session(username)` — зберігає JSON-токен у `data/session.json`
- `load_session()` — завантажує та перевіряє токен (TTL 30 днів)
- `clear_session()` — видаляє токен при виході

#### `taskdb.py` / `workersdb.py` / `usersdb.py`
CRUD-функції для відповідних сутностей. Кожна функція відкриває з'єднання, виконує запит і закриває з'єднання. Підтримують фільтрацію, пошук і сортування на рівні SQL.

---

### `login/` — авторизація

#### `lock.py`
Захист від підбору пароля:
- Зберігає лічильник невдалих спроб у таблиці `login_attempts`
- Після **3 невдалих спроб** — блокування на **5 хвилин**
- `is_locked(username)` → `(bool, seconds_remaining)`
- `record_failed(username)` / `reset_attempts(username)`

#### `login.py`
`LoginWindow` — вікно входу. Підтримує:
- Автоматичний вхід через збережену сесію (після `after(150ms)`)
- Checkbox «Запам'ятати пристрій»
- Відображення кількості залишених спроб
- Посилання на FAQ без входу в систему

---

### `main/` — головне вікно

#### `main.py`
`CRMApp` — кореневий клас застосунку:
- Ліво-права розмітка: сайдбар (`self.side`) + контент (`self.body`)
- Навігаційні кнопки з підсвіткою активної сторінки
- Метод `show(key)` — роутер між сторінками
- Годинник у хедері, що оновлюється щосекунди (`_update_clock`)
- Таймер (`_tick`) — оновлює лічильник незалежно від поточної сторінки

#### `theme.py`
Два словники `THEMES['light']` і `THEMES['dark']` з усіма кольорами інтерфейсу. Глобальна змінна `C` — посилання на поточну тему. Перемикання теми перебудовує весь UI через знищення та повторне створення дочірніх віджетів.

#### `uihelper.py`
Фабрики UI-компонентів і кастомні графіки на `Canvas`:

| Компонент | Опис |
|---|---|
| `mk_btn()` | Кнопка без рамки, з курсором-рукою |
| `mk_entry()` | Поле введення з акцентною рамкою при фокусі |
| `mk_label()` | Підпис із підтримкою розміру та жирності |
| `make_tree()` | `ttk.Treeview` зі скролами, чергуванням рядків і кольоровими тегами |
| `PieChart` | Кругова діаграма з легендою та відсотками |
| `BarChart` | Стовпчаста діаграма з підписами осей |
| `DonutChart` | Діаграма-пончик з текстом у центрі |

Всі графіки — нативний `tk.Canvas`, без зовнішніх бібліотек. Автоматично перемальовуються при зміні розміру вікна.

#### `dashboard.py`
Головний дашборд зі скролом:
- Привітання залежно від часу доби
- 6 KPI-карток з кольоровими акцентами та іконками
- Блок діаграм: статуси завдань (donut), статуси працівників (donut), відділи (bar)
- Фінансовий блок та бар-чарт пріоритетів
- Блок «Мої завдання» — задачі, де відповідальний = поточний користувач

#### `taskpage.py`
Сторінка завдань з двома вкладками:
- **Список** — таблиця з пошуком, фільтром за пріоритетом/статусом, масові дії
- **Аналітика** — donut за пріоритетами, bar за статусами

#### `faq.py`
Інтерактивна документація: ліва панель з кнопками-розділами (скролюється), права — вміст. Доступна без входу в систему через кнопку на екрані логіну.

---

### `people/` — персонал

#### `workerpage.py`
Сторінка «Працівники» з трьома вкладками:
- **Список** — пошук за ім'ям/посадою/відділом, фільтр за статусом, кнопки адміна
- **Статистика** — таблиця по відділах: кількість, сер. зарплата, фонд ЗП, відсотки
- **Графіки** — 4 графіки: розподіл по відділах (pie), зарплата (bar), статуси (pie), фонд ЗП (bar)

Форма додавання/редагування (`_worker_form`) — використовує `Modal` зі скролом, тому кнопка «Зберегти» **завжди доступна**.

#### `profile.py` / `profilepage.py`
- `ProfileWindow` — перший екран після входу, дозволяє заповнити телефон/email/організацію та одразу змінити пароль
- `profilepage.py` — той самий діалог для виклику з кнопки «Профіль» у сайдбарі

---

### `timer/` — облік часу

`TimerPage` з трьома вкладками:
- **Таймер** — великий цифровий годинник (`Courier New 56px`), кнопка старт/стоп, загальний підсумок
- **Мої сесії** — таблиця останніх 50 сесій із часом початку, кінця та тривалістю
- **Статистика** — 3 KPI-картки (загальний час, кількість сесій, середня тривалість) + bar-chart останніх 10 сесій

---

### `admin/` — адміністрування

`AdminPage` з чотирма вкладками (доступна тільки ролі `admin`):

| Вкладка | Можливості |
|---|---|
| **Користувачі** | Створення, зміна ролі, видалення, скидання блокування |
| **Працівники** | Повний CRUD без обмежень |
| **Завдання** | Повний CRUD, масові зміни статусів |
| **Аналітика** | Час роботи всіх, рейтинг продуктивності (з медалями 🥇🥈🥉), bar-chart по юзерах, таблиця сесій |

---

### `log/` — журналювання

`log_action(action)` — єдина функція, що пишеться скрізь:
1. Виводить у `logging` (консоль)
2. Дописує рядок у `logs/crm_log.txt`
3. Вставляє запис у таблицю `logs` (timestamp, username, action)

Звичайний користувач бачить тільки свої записи; адмін — всі.

## База даних

Файл: `db/crm.db` (SQLite 3). Другий файл `crm.db` у корені — резервна копія або legacy, основна БД у `db/`.

### Схема

```sql
-- Користувачі системи
CREATE TABLE users (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    username     TEXT UNIQUE NOT NULL,
    password     TEXT NOT NULL,           -- SHA-256 хеш
    role         TEXT DEFAULT 'user',     -- 'user' | 'admin'
    phone        TEXT DEFAULT '',
    email        TEXT DEFAULT '',
    org          TEXT DEFAULT '',
    created      TEXT,                    -- ISO 8601
    locked_until TEXT DEFAULT '',
    last_login   TEXT DEFAULT ''
);

-- Лічильник невдалих спроб входу
CREATE TABLE login_attempts (
    username     TEXT PRIMARY KEY,
    count        INTEGER DEFAULT 0,
    last_attempt TEXT
);

-- Співробітники компанії
CREATE TABLE workers (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    name       TEXT NOT NULL,
    department TEXT DEFAULT '',
    salary     REAL DEFAULT 0,
    hire_date  TEXT DEFAULT '',
    phone      TEXT DEFAULT '',
    email      TEXT DEFAULT '',
    position   TEXT DEFAULT '',
    note       TEXT DEFAULT '',
    status     TEXT DEFAULT 'Активний'   -- Активний | Звільнений | Відпустка | Лікарняний
);

-- Завдання
CREATE TABLE tasks (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    title     TEXT NOT NULL,
    assignee  TEXT DEFAULT '',
    priority  TEXT DEFAULT 'Середня',    -- Критична | Висока | Середня | Низька
    deadline  TEXT DEFAULT '',
    status    TEXT DEFAULT 'Активне',    -- Активне | В роботі | На паузі | Виконано
    author    TEXT DEFAULT '',
    created   TEXT,
    note      TEXT DEFAULT '',
    category  TEXT DEFAULT ''
);

-- Журнал дій
CREATE TABLE logs (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT,
    username  TEXT,
    action    TEXT
);

-- Сесії таймера робочого часу
CREATE TABLE time_sessions (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    username   TEXT,
    started    TEXT,
    ended      TEXT,
    duration   INTEGER                   -- секунди
);

-- Особисті нотатки
CREATE TABLE notes (
    id        INTEGER PRIMARY KEY AUTOINCREMENT,
    username  TEXT,
    title     TEXT DEFAULT '',
    content   TEXT DEFAULT '',
    color     TEXT DEFAULT '#FFF9C4',
    created   TEXT,
    updated   TEXT
);
```

### Міграції

При кожному запуску `db_init()` перевіряє наявність нових колонок через `PRAGMA table_info()` і додає їх якщо відсутні — це дозволяє оновлювати схему без ручного SQL і без втрати даних.

## Безпека

- **Паролі** — зберігаються тільки як SHA-256 хеш, оригінал ніде не фіксується
- **Брутфорс** — 3 невдалі спроби → автоблокування на 5 хвилин; адмін може скинути вручну
- **Сесія** — JSON-файл з міткою часу; TTL 30 днів; видаляється при явному виході
- **Самовидалення** — адмін не може видалити власний акаунт
- **Ізоляція журналу** — звичайний user бачить лише свої записи
- **WAL режим** — SQLite `PRAGMA journal_mode=WAL` запобігає пошкодженню БД при аварійному завершенні

### Резервне копіювання

Вся інформація зберігається у єдиному файлі `db/crm.db`. Для резервної копії достатньо скопіювати цей файл (закрийте програму перед копіюванням):

```bash
cp db/crm.db backups/crm_$(date +%Y%m%d).db
```

## Розробка

### Додавання нової сторінки

1. Створіть модуль у відповідній папці (або у `main/`)
2. Додайте кнопку навігації у `main/main.py` → `_sidebar()` → список `pages`
3. Зареєструйте роут у `show()` → словник `dispatch`
4. Додайте заголовок у словник `info`

### Додавання нової колонки до БД

Додайте в `db/db.py` → `db_init()`:

```python
existing = [r[1] for r in conn.execute('PRAGMA table_info(your_table)').fetchall()]
if 'new_column' not in existing:
    conn.execute('ALTER TABLE your_table ADD COLUMN new_column TEXT DEFAULT ""')
```
